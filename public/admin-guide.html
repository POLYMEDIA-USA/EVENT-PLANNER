Admin Guide

FunnelFlow

Full administrator reference — complete platform management

As an Admin, you have full access to every feature in FunnelFlow. You can manage events, leads, invitations, email campaigns, QR check-ins, interactions, reports, user accounts, and system settings.

Login URL: https://your-app-url.run.app

Signing In

  1. Navigate to the app URL.
  2. Enter your Email / Username and Password.
  3. Click "Sign In".
🔑
The default admin account is admin / admin123!. Change this password immediately after first login by having another admin update it in the Users page.

Events

◈

Create & Manage Events

Admin only — Supervisors & Sales Reps have read-only access
  1. Click "Create Event" to open the form.
  2. Fill in Name, Date, Time, Location, and optional Description.
  3. Click "Create".

From the events list you can:

  • Edit — Update event details
  • Set Active / Inactive — Toggle event status
  • Delete — Permanently remove an event
📌
Only active events appear in the Dashboard event selector and are used for email campaigns.

Leads

◉

Lead Management

Admins see all leads across all organizations

Leads represent potential event attendees. Admins see leads from every organization.

Adding a Lead

  1. Click "Add Lead".
  2. Fill in: Full Name, Title, Company, Email, Phone, Alt Email.
  3. Click "Create".

Managing Leads

  • Search — Filter by name, email, company, phone, input by, or assigned rep
  • Edit — Update lead details including assigned rep
  • Delete — Remove a lead permanently

Input By & Assigned Rep (v0.3.0)

Every lead tracks two key fields:

  • Input ByAuto-populated with the name and organization of the user who created the lead. Cannot be changed after creation.
  • Assigned RepA dropdown picker showing all registered users (any role). Select the sales rep responsible for this lead. Editable at any time.

Bulk Import from File (v0.3.0)

  1. Click "Import from File" on the Leads page.
  2. Upload a CSV or XLSX file.
  3. Map file columns to lead fields. Full Name supports combining multiple columns (e.g., "First Name" + "Last Name").
  4. Select an Event to assign all imported leads to (required).
  5. Review, filter, and deselect rows as needed.
  6. Click "Import". All imported leads are assigned status "possible" and the Input By field records the importing user.
📁
Multi-column name mapping: If your file has separate "First Name" and "Last Name" columns, add both to the Full Name mapping. They'll be joined with a space. A live preview shows the combined result.

QR Scanner

Use the QR Scanner page to check in attendees at events.

  1. Click "Start Scanner" to activate your device camera.
  2. Point the camera at the attendee's QR code (received in their confirmation email).
  3. The system automatically reads the code and displays the attendee's name, title, and organization.
  4. Optionally add interaction notes and click "Save Notes".

Manual check-in: If the camera can't read the QR code, the attendee can provide their 4-character check-in code (e.g., K7PN — shown in their confirmation email and RSVP page). Enter it in the text field and click "Check In".

📱
QR scanning works on all modern browsers (Chrome, Safari, Firefox). Best on mobile devices with rear-facing cameras.

Interactions

The Interactions page is a customer-centric note-taking hub. Admins see interactions from all sales reps across all organizations.

Customer List (Left Panel)

  • Browse all customers with their interaction count and RSVP status
  • Customers with existing interactions appear first
  • Search by name, title, company, or email
  • Click a customer to view their interactions

Interaction Detail (Right Panel)

  • Customer header — name, title, company, email, phone, and RSVP status
  • Add a Note — type your notes and click "Save Note". Your name is recorded automatically.
  • Interaction History — all notes, QR check-ins, RSVP events from every user, newest first
👥
Multiple team members can log notes on the same customer. Each note shows who wrote it and when, giving the full engagement history.

Reports

Overview Tab

Organization-level statistics and a sortable master table of all leads with their statuses, RSVP responses, and interaction counts.

Email Log Tab

Complete log of all outbound emails sent through the platform. Features include:

  • Search — Filter by recipient, subject, or type
  • Preview — Click any email to see a PDF-style preview of the full HTML email
  • Print — Print individual email previews for records

Post-Event Tab

After an event concludes, use the Post-Event tab to review attendees, their interactions, and distribute reports to sales reps.

  • Attendee list — All checked-in attendees with interaction counts and which reps engaged them
  • Expandable detail — Click any attendee to see their full interaction history (notes, timestamps, rep names)
  • Rep selection panel — Choose which sales reps should receive a report. Each rep's report includes only the attendees they interacted with.
  • Email Reports — Send formatted HTML reports to selected reps with attendee details and all interaction notes
  • Download CSV — Export the complete event dataset (name, title, company, email, phone, status, interactions, notes summary, reps involved)
📊
Tip: Use "Select all reps with interactions" to quickly pick everyone who engaged with attendees, then review and deselect as needed.

Only visible to Admin users

  2. Fill in: Full Name, Email/Username, Phone, Password, Organization.
  3. Select a Role:

⚙

Platform Configuration

Only visible to Admin users

General Settings

  • Company NameDisplayed in email headers and branding
  • Logo URLURL to your company logo (used in emails)
  • App URLYour deployed app URL — used for RSVP links in emails

Email (SMTP) Settings

  • SMTP Host*e.g., smtp.gmail.com
  • SMTP Port587 (TLS) or 465 (SSL) — defaults to 587
  • SMTP User*Your email address used for sending
  • SMTP Password*App Password (not your regular password)
  • From AddressOptional — defaults to SMTP user if blank
📧
Google Workspace / Gmail: Use smtp.gmail.com, port 587, and generate an App Password. You must have 2-Step Verification enabled on the sending account.

Click "Save Settings" after making changes. A green confirmation message appears on success.

Lead Scoring

Automatic hot / warm / cold classification

Every lead is automatically scored based on:

  • Interaction count (+5 per interaction, max 25)
  • Emails sent (+5 per email, max 15)
  • RSVP accepted (+20 points)
  • Event attended (+30 points)
  • Has assigned rep (+5 points)

Labels: Cold (0-19), Warm (20-49), Hot (50+). Scores appear in the Leads table and on dashboards.

Pipeline & Funnel Dashboard

Visual conversion rates between stages

The Dashboard now shows a visual pipeline funnel: Possible → Invited → Accepted → Declined → Attended, with bar charts and conversion rates between each stage. Also shows Lead Temperature distribution (Hot/Warm/Cold counts).

Follow-Up Tasks

Assign, track, and manage post-event follow-ups

Create tasks for leads with due dates, priorities (low/medium/high), and assignees. Track status (pending/completed/overdue).

  1. Navigate to Tasks in the sidebar.
  2. Click "Add Task".
  3. Select a customer, assign to a rep, set due date and priority.
  4. Add a title and description, then save.

Filter by: All, My Tasks, Overdue, or Completed. Overdue tasks are color-coded red.

Tips & Best Practices

  • Set up SMTP first. Email campaigns won't work until SMTP settings are configured in Settings.
  • Set the App URL. RSVP links in emails need the correct app URL to function properly.
  • Use smart selection buttons when sending emails to avoid duplicate sends.
  • Send confirmations to accepted RSVPs. Confirmation emails include the QR code needed for event check-in.
  • Review the email log in Reports to verify what was sent and troubleshoot delivery issues.
  • Create events before leads. Leads can be associated with events for better tracking.
  • Use Supervisor role for team leads who need full operational access but shouldn't manage system settings or user accounts.
